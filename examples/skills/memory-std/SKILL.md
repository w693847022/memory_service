# Memory-std 技能

根据用户输入，将项目规范记录到 memory_mcp。

## 触发方式

```
/memory-std <输入内容>
```

## 流程概览

```
确定项目 → 解析输入类型 → 探索/拆分 → 记录标签并添加
```

## 输入类型与处理

### 快速选择规范方向

如果用户未提供具体输入，展示以下选项供选择：

```
请选择要记录的规范方向：
  1. 代码规范   - 代码风格、缩进、格式化
  2. 命名规范   - 变量/函数/类命名
  3. Git规范    - 提交信息、分支管理
  4. API规范    - 接口设计、返回格式
  5. 测试规范   - 单元测试、集成测试
  6. 文档规范   - 文档结构、注释规范
  7. 部署规范   - 部署流程、CI/CD
  8. 目录结构   - 项目组织、文件命名
  9. 安全规范   - 认证、授权、加密
 10. 性能规范  - 性能指标、优化原则

输入编号或直接描述规范内容
```

### 输入类型处理

- **无输入**: 展示规范方向选项供选择
- **方向编号**: `1`, `2` 等 → 按该方向探索
- **方向名称**: `代码规范`, `测试规范` → 按该方向探索
- **代码范围**: `src/api`, `tests/` → 探索该范围内所有规范种类
- **确切规范**: `使用snake_case`, `4空格缩进` → 拆分提取关键词，按种类归类后记录

## 格式要求

- **summary**: `{种类}规范-{关键词}`
- **content**: 规范详情

## 阶段 1: 确定项目

```python
project_list()
```
> 自动匹配当前目录项目

## 阶段 2: 解析输入类型

### 类型判断

```
输入内容 → 判断类型 → 确定探索范围
```

- **无输入**: 展示规范方向选项，等待用户选择
- **方向编号**: `1`-`10` → 对应规范方向标签表中的方向
- **方向名称**: 匹配规范方向标签表（如 `代码规范`、`测试规范`）→ 按该方向探索
- **代码范围**: 含路径分隔符 `/` 或常见目录名 → 探索该目录下的所有规范种类
- **确切规范**: 其他情况 → 拆分提取关键词，归类后记录

### 规范方向标签

| 规范方向 | 标签 | 说明/示例 |
|---------|------|----------|
| **代码规范** | code | 代码风格、缩进、格式化、语言惯用法 |
| **命名规范** | naming | 变量/函数/类命名规范 |
| **Git规范** | git | 提交信息格式、分支管理 |
| **API规范** | api | 接口设计、返回格式、错误处理 |
| **测试规范** | testing | 单元测试、集成测试、测试命名 |
| **文档规范** | docs | 文档结构、注释规范、README |
| **部署规范** | deploy | 部署流程、环境配置、CI/CD |
| **目录结构** | structure | 项目目录组织、文件命名 |
| **安全规范** | security | 认证、授权、数据加密 |
| **性能规范** | performance | 性能指标、优化原则 |
| **项目规范** | project | 其他项目级规范（兜底） |

---

## 阶段 3: 项目内探索

### 规范方向探索

按选择的规范方向探索项目中已有或应有的关联规范。

### 代码范围探索

若指定代码范围（如 `src/api`），则探索该目录下的所有规范种类。

### 拆分规范

```
输入: 使用snake_case命名

拆分结果:
  - 方向: naming
  - 关键词: snake_case
  - summary: 命名规范-snake_case
  - content: 使用snake_case进行变量/函数命名
```

---

## 阶段 4: 记录

### 标签处理

```python
project_tags_info(project_id)
# 注册未存在的标签
tag_register(project_id, tag_name, summary)
```

### 添加标准

```python
project_add(
    project_id="<项目ID>",
    group="standards",
    summary="{种类}规范-{关键词}",
    content="<规范详情>",
    tags="<种类标签>"
)
```
